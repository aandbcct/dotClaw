**Session**：一个连续的会话窗口

**AgentRun**：一条运行记录的元数据、终态、错误、关联 session/agent、事件序列；

**Event Log**：`run_started`、状态迁移、LLM 调用结束、工具开始/结束、审批、取消、失败、完成等结构化事实，按发生顺序追加；

**Checkpoint**：仅为恢复服务，保存可恢复所需的 `AgentState`、本轮临时消息、下一动作、预算和未完成工具信息；

**Trace**：订阅事件生成的调试/展示产物，可删、可重建，不能反过来被 Runtime 读取；

**Conversation**：只在成功提交时投影出最终 assistant message。

| 容器           | 回答的问题                        | 记录内容                                                     | 不记录内容                             |
| -------------- | --------------------------------- | ------------------------------------------------------------ | -------------------------------------- |
| `Conversation` | 用户与 Agent 已经成功交流了什么？ | 用户输入、成功完成的最终助手回复                             | 工具调用、失败、取消、审批、状态机细节 |
| `AgentRun`     | 这次执行是谁、何时、因何结束？    | `run_id`、agent/session、起止时间、终态、最终消息引用、错误摘要、事件范围 | 完整事件副本、完整上下文、恢复状态     |
| `RunEvent`     | 运行过程中客观发生了什么？        | 有序状态迁移、LLM/工具/审批/取消/错误等结构化事实            | 可恢复状态快照、面向用户的会话语义     |
| `Checkpoint`   | 中断后从哪里、安全地如何继续？    | `AgentState` 快照、临时消息、待执行动作、预算、恢复游标      | 审计全历史、Trace 格式化文本           |
| `Trace`        | 开发者或用户如何理解这次运行？    | 从 `RunEvent` 渲染出的时间线、摘要、调试视图                 | 任何执行或恢复的唯一真相               |
