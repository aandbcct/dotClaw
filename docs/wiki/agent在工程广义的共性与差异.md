`Agent`在广义上我们想象成一个和人一样具有记忆、技能、执行能力的实体。但`Agent` 在工程上是轻量的“身份化交互门面/句柄”，可以随时创建，不是需要一对一常驻的执行器。一次真正执行任务的实体是 `Run`，不是长驻的 `Agent` 对象。

可以这样理解：

| 概念     | 广义 AI 语境                       | 工程中的精确定义                                      | 生命周期                 |
| -------- | ---------------------------------- | ----------------------------------------------------- | ------------------------ |
| Agent    | 能理解任务、调用工具、行动的智能体 | `AgentIdentity` + 面向该身份的轻量门面                | 配置长期存在；门面可重建 |
| Identity | 人设、能力、角色                   | `agent_id`、模型、提示词、工具白名单、上下文策略      | 配置版本化、可持久化引用 |
| Runtime  | Agent 的“运行环境”                 | 无身份的共享执行内核、状态机、恢复、审批、并发协调    | ApplicationHost 生命周期 |
| Run      | 一次 Agent 工作                    | 单次请求的状态机、消息、事件、checkpoint              | 一次任务生命周期         |
| Session  | 一段连续对话                       | 历史、摘要、会话级上下文，以及“默认使用哪个 identity” | 用户会话生命周期         |
